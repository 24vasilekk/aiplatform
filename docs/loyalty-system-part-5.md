# Loyalty System (Part 5)

## Аудит текущей архитектуры

1. Платежи идут через `PaymentIntent` + `Wallet` c идемпотентностью по `(userId, idempotencyKey)`.
2. Прогресс учебы агрегируется через `buildUserProgressSnapshot`, где можно достоверно определить завершение курса.
3. Денежные транзакции уже реализованы ledger-подходом (`WalletTransaction`), поэтому для лояльности выбран отдельный ledger и bucket-модель, чтобы корректно учитывать срок жизни баллов.

## Правила программы

1. Награда за завершение курса: `1200` баллов.
2. Срок жизни начисленных баллов: `180` дней.
3. Конвертация в скидку: `1 балл = 1 копейка` (`0.01 RUB`).
4. Ограничение скидки: не более `30%` от суммы заказа.
5. Минимальная сумма заказа для скидки: `500 RUB` (`50_000` копеек).
6. Минимальная сумма к оплате после скидки: `100 RUB` (`10_000` копеек).
7. Лимит баллов на 1 заказ: `70_000` баллов.
8. Списание выполняется из ближайших к истечению bucket-ов (FIFO по `expiresAt`), чтобы минимизировать потери пользователя.

## Ограничения и надежность

1. Начисление за курс идемпотентно: ключ `course_completion:<courseId>`.
2. Применение скидки идемпотентно: `wallet_loyalty_redeem:<paymentIdempotencyKey>`.
3. Если оплата кошельком не проходит по причине недостатка средств, списание баллов откатывается компенсирующей транзакцией `DISCOUNT_ROLLBACK`.
4. Истечение баллов происходит lazily перед чтением/списанием, без отдельного cron.
