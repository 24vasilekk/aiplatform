# Часть 3: Аналитика обучения и воронка

## Цель

Добавить измеримость реального обучения, а не только auth/blog-метрик:

- активные ученики (DAU/WAU),
- прогресс по урокам,
- результативность по заданиям,
- интенсивность использования AI.

## Архитектурные решения

1. **Единый поток событий через `AnalyticsEvent`**
   - Расширены события: `lesson_progress_updated`, `task_checked`, `ai_chat_message`, `ai_solution_analyzed`.
   - События пишутся прямо в критические API-роуты (уроки, задания, чат, AI-анализ).

2. **Снимки воронки для админки**
   - Новый backend-слой `getLearningAnalyticsSnapshot(days)` строит KPI-срез за период:
     - регистрации,
     - активные пользователи,
     - completed/progressed уроки,
     - attempted/correct tasks,
     - AI chat messages, AI analyses,
     - оплаты и конверсия.

3. **Дневные агрегаты для трендов**
   - `DailyMetricAggregate` хранит ежедневные продуктовые метрики и ретеншн-кохорты D1/D7.
   - `ensureDailyMetricAggregates(days)` заполняет недостающие дни автоматически.

## Что добавлено в продукте

- Админский виджет обновлен до **Часть 2-4**:
  - каналы регистраций + блог (Часть 2),
  - KPI обучения/AI (Часть 3),
  - таблица дневных агрегатов (база для Часть 4).
- Новый админ API:
  - `GET /api/admin/analytics/summary?days=30`
